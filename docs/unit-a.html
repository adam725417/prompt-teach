Prompt 五要素

Prompt 不是咒語，是工程設計。五個要素缺一不可，每個都有明確的工程意義。

整合範例：包含五要素的完整 Prompt

⭐ 建議寫法 
[目標 Goal]
你是工廠設備工程師，請分析以下機台異常訊息，判斷嚴重程度與可能原因。

[資料 Context]
機台ID: WE-27
班別: 早班
異常訊息: 收線張力波動超過 ±15%，已連續發生 3 次
最近維修紀錄: 2週前更換收線滾輪

[規則 Constraints]
- 若資料不足，請回覆「資料不足，無法判斷」，不可捏造
- 可能原因最多列 3 項，按機率排序
- 使用繁體中文回覆

[格式 Output Format]
以 JSON 格式輸出：
{"summary": "50字內摘要", "severity": "高/中/低", "possible_causes": [], "action": "建議行動"}

[評估 Evaluation]
好的回應：格式正確、摘要精準、原因具體可操作
差的回應：含糊描述、超出3項原因、格式不符
教學提示：同一句話，放在不同角色位置，效果可以差很多！先看角色卡片，再做拖拉分類練習。

Prompt 角色結構

在實務上，Prompt 不只是「一段文字」，而是由三種角色訊息組成的結構

同題比較實驗：只改角色分配，看輸出差異

任務相同：設備異常摘要。只是訊息放的「位置」不同，結果天差地遠。
